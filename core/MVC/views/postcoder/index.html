Postcoder
Introduction
Welcome to the postcoder api. The purpose is to provide you with an easy to use, free and open source method to lookup postcodes in the UK and return information. The api provides locality information, local streets, local transport and much more. You can chose how little or how much information to get back.
The data and project
The data sets used in this project are hosted on
  • github https://github.com/ouchiko/ukord-postcodes. These are the raw mysql inserts for creating a complete database replica of the data presented here. The interface, my php framework and logic for the API are also available at
  • github https://github.com/ouchiko/wcframework. If you're interested in the API element then the MVC/models/PostCodeModel.php and MVC/controllers/PostcoderController.php are the two elements to look at.
    Using the API
    The following is an example of a basic call to the API. You should note that the postcode can be partial or full and the system will return all the information for that range of postcodes. Note that the service is limited to returning data with a minimum of three charatcers.
  • Requested a basic postcode
    http://{{ vars.server.server_name }}/postcoder/api/[postcode]/index.html
    Now, it's 99% likely you want that data returned in a more computer friendly way. Just switch the .html to either .printr, .json or .ser. Each of these will allow you to easily read in the data into your application. The .html extension is purely for a human readable form of the data.
    When making a request you can define the level of information you want. The variable choice can be added with any combination of the following options:
  • Types of data request
    ?choice=streets,postcodes,travel
    When we put all that together we can format a request for the data in .json format and include all the information we have on streets, postcodes and any local travel information
  • Defining the information you want
    http://{{ vars.server.server_name }}/postcoder/api/[postcode]/index.json?choice=streets,travel,postcodes
    Data responses
